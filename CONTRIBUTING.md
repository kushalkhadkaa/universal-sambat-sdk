# Contributing to Nepali DatePicker Studio

Thank you for your interest in contributing to Nepali DatePicker Studio! As an open-source project, contributions from developers like you help maintain high performance, styling quality, and calendrical accuracy.

---

## Code of Conduct
Please be polite and respectful in all issues, pull requests, and discussions.

---

## How to Contribute

### 1. Reporting Bugs & Layout Issues
- Search existing open issues before filing a new bug report.
- Provide step-by-step instructions, browser versions, and configuration options used to replicate the layout or javascript error.

### 2. Suggesting Features
- Outline your proposed feature, use-cases, and how it aligns with the lightweight, zero-dependency design of the library.

### 3. Submitting Pull Requests
- Fork the repository.
- Create a clean feature branch: `git checkout -b feature/amazing-feature`.
- Keep modifications highly focused (avoid unrelated code formatting churn).
- Ensure your changes work locally and do not introduce console warnings or javascript errors.
- Commit clean, descriptive messages.
- Submit a Pull Request targeting the `main` branch.

---

## Coding Style Guidelines
- **Zero Dependencies**: Do not introduce external libraries (e.g. jQuery, React, Moment.js). Everything must remain vanilla ES5/ES6 JS and plain CSS.
- **Indentation**: Use spaces (2 spaces) instead of tabs.
- **Accuracy Rules**: Calendar date lookup matrices or mapping modifications must be backed by official astronomical resources or government declarations. Do not modify translation algorithms without verified references.

---

## Security Vulnerabilities
If you discover a security vulnerability, please do not open a public issue. Email the details directly to **kushalkhadkaa@gmail.com** instead, so we can release a patched version immediately.

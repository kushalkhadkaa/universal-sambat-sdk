API Documentation

Complete reference guide for installing, configuring, and extending the Nepali DatePicker Studio library.

Introduction #

Nepali DatePicker Studio is a lightweight, dependency-free JavaScript calendar widget designed explicitly for selecting Bikram Sambat (BS) dates on modern web pages. It features an advanced bidirectional AD-to-BS conversion engine, holiday mapping markers, responsive theme layouts, and robust keyboard navigation.

This library is built with pure Vanilla ES5 JavaScript and CSS, requiring zero preprocessors, zero npm build processes, and zero database storage. It acts as a drop-in asset for standard HTML forms.

How it Works #

At its core, Nepali DatePicker Studio transforms standard HTML input fields into interactive, touch-friendly, and accessible calendar drop-down panels. It works using the following simple lifecycle:

1

DOM Binding

The library targets your input element (using CSS selectors like #datepicker) and attaches listeners for focus, click, and keyboard inputs.

2

Floating Calendar Generation

When clicked, the picker dynamically appends a calendar container (.ndp-container) positioned relative to the target input. It renders the month grid based on current Bikram Sambat year and month data.

3

Bidirectional Conversion

If Gregorian date labels are enabled (showAdDate: true), the engine converts individual BS grid dates to AD equivalents in real-time. Conversely, if you initialize with a default value, it calculates the equivalent BS grid placement.

4

Input Synchronization

Clicking a date closes the popup, formats the selected date (e.g. २०८३-०३-१५), populates the text field, and fires custom developer callbacks (onChange).

BS vs AD Calendrical Logic #

1. What is Bikram Sambat (BS)?

Bikram Sambat (BS) is the official historical calendar used in Nepal. It is a solar-sidereal calendar based on ancient Hindu astronomy. The BS calendar is approximately 56.7 years ahead of the Western Gregorian calendar (AD).

2. Logically, how do BS and AD differ?

Unlike the Gregorian calendar, which has a rigid, fixed number of days per month (e.g., January always has 31 days, September always has 30 days), month lengths in Bikram Sambat are variable and change from year to year.

For instance, the first month of the Nepali year, Baisakh, might have 31 days in one year, 30 days in the next, or even 32 days in another!

Why are BS month lengths variable?

The Bikram Sambat calendar is tied to the Sun's actual sidereal transit through the 12 zodiac signs (Rashis). Since the earth's orbit is elliptical, the Sun travels at varying speeds across the sky (described by Kepler's laws of planetary motion). The number of days in a BS month represents the exact number of days the Sun takes to traverse a specific zodiac constellation, which astronomical computations calculate as varying between 29 to 32 days.

3. The Core Conversion Formula & Reference Epoch

Because BS month lengths are astronomical and variable, there is no simple mathematical formula (like leap-year calculations in AD) that can convert AD to BS directly without access to Zodiac transit tables.

Instead, the library establishes a known anchor point (the **Reference Epoch**) and utilizes a validated database lookup table containing month lengths for years 1970 BS through 2100 BS.

BS to AD Logic

  1. Establish the Base Anchor: Baisakh 1, 1970 BS maps exactly to April 13, 1913 AD.
  2. Calculate the absolute delta days (\(D_{\Delta}\)) from Baisakh 1, 1970 BS to your selected target BS date using the lookup table.
  3. Add \(D_{\Delta}\) to April 13, 1913 AD using standard Gregorian date arithmetic to return the correct Gregorian date object.

AD to BS Logic

  1. Calculate the total days difference (\(D_{\Delta}\)) between the target AD date and the base AD anchor (April 13, 1913).
  2. Traverse forward from year 1970 BS, month by month, subtracting the number of days in each month from \(D_{\Delta}\).
  3. When the remaining days are less than the days in the current target month, the traversal stops. The current year, month index, and remaining days represent your target BS date.

4. Mathematical Representation

Let the BS epoch start at year \(Y_0 = 1970\) and day \(D_0 = 1\) (representing Baisakh 1, 1970 BS). Let the AD epoch anchor date be \(A_0 = \text{April 13, 1913 AD}\).

To convert a target Bikram Sambat date \((Y_{\text{target}}, M_{\text{target}}, D_{\text{target}})\) to its Gregorian equivalent, the total elapsed days \(D_{\Delta}\) since the epoch is computed as:

\[ D_{\Delta} = \sum_{y=1970}^{Y_{\text{target}}-1} \text{DaysInYear}(y) \;+\; \sum_{m=1}^{M_{\text{target}}-1} \text{DaysInMonth}(Y_{\text{target}}, m) \;+\; (D_{\text{target}} - 1) \]

Once the total days difference \(D_{\Delta}\) is computed, the equivalent Gregorian Date is calculated using basic millisecond epoch addition:

\[ \text{Gregorian Date} = A_0 + (D_{\Delta} \times 24 \times 60 \times 60 \times 1000 \text{ ms}) \]

This mathematical arithmetic enables perfect, single-millisecond-precision conversions across the entire supported range of calendar dates.

Step-by-Step Setup #

Integrating Nepali DatePicker Studio into a web application is simple. Follow this step-by-step guide to get a basic working calendar input in under 2 minutes.

Basic Checklist:

  1. Confirm you are loading the stylesheet in the <head> of your page.
  2. Provide a standard text input field with a target selector (e.g., class or ID).
  3. Load the JavaScript bundle at the bottom of the page.
  4. Instantiate the class inside a standard DOM event listener to avoid loading sequence errors.

Basic Usage #

Once the library script has finished loading, select your input field element and initialize a new NepaliDatePicker class instance using standard CSS selectors or direct DOM nodes.

JavaScript Initialization 
document.addEventListener('DOMContentLoaded', function() {
  const picker = new NepaliDatePicker('#my-datepicker-input', {
    theme: 'glassmorphism',
    lang: 'ne', // Set language (ne = Nepali, en = English)
    showAdDate: true, // Display corresponding AD date below BS day
    onChange: function(date) {
      console.log('Selected date details:', date);
      /*
        Returns object structure:
        {
          year: 2083,
          month: 3,
          day: 15,
          formatted: "२०८३-०३-१५",
          adDate: Date object (mapping to 2026-06-29)
        }
      */
    }
  });
});

How the UI Looks & Behaves

When initialized, the picker renders a floating, pixel-perfect modal panel. Here is a description of its structural components:

  • Header Controls: Contains month/year quick jump select menus (e.g. Asar 2083) along with beautiful, responsive chevron navigation arrows.
  • Weekday Grid: Shows week header labels matching the language option (e.g. आईत, सोम, मंगल... for Nepali, or Su, Mo, Tu... for English).
  • Active Day Grid: Displays days in Devanagari or English digits. Current today dates feature a pulsing border, weekends are styled with distinct red colors, and hovered cells show a smooth glow transit backdrop.
  • Mini AD Labels: Tiny, secondary English dates (e.g. 29) render directly underneath each BS day number, offering immediate AD clarity.
  • Control Footer: Features a quick-action Today (आज) link on the left, and a red-accented Clear (हटाउनुहोस्) action button on the right.

Best Practices & Tips #

To ensure optimal user experience, database consistency, and front-end performance, follow these industry best practices when integrating Nepali DatePicker Studio:

1. Standardize Server-Side Submission Format

While users like seeing formatted Devanagari numbers (e.g., २०८३-०३-१५), relational databases require standard English alphanumeric dates (YYYY-MM-DD).
Solution: Use the exportAdInput configuration option to automatically synchronize and map selected BS dates to a hidden AD input field that submits directly to your backend.

2. Avoid Memory Leaks in SPAs (React, Vue, Angular)

In Single Page Applications where component layouts load and destroy dynamically, event listeners can remain orphaned in the global window object.
Solution: Capture the instance and call the .destroy() method when your component unmounts: 

// React Component Unmount Example
useEffect(() => {
  const picker = new NepaliDatePicker('#react-input');
  return () => picker.destroy();
}, []);

3. Leverage Date Boundaries (UX Constraints)

Prevent users from selecting illogical date values (like future dates for a birthdate, or past dates for booking an event).
Solution: Use constraints like futureOnly: true, pastOnly: true, or custom boundary properties: 

new NepaliDatePicker('#dob', {
  pastOnly: true, // Block future dates
  maxDate: { year: 2083, month: 3, day: 1 }
});

4. Theme Overrides via CSS Variables

Avoid hacking the core minified stylesheet. It is much safer to override the root CSS custom properties in your own stylesheet or inside parent container elements.
Example: 

.my-form-container {
  --ndp-primary: #ff007f; /* Custom color */
  --ndp-border-radius: 8px; /* Custom shape */
}

5. Mitigate Timezone Shifts in Backend Sync

Nepal Standard Time (NST) operates at GMT+5:45. If you convert BS dates to Gregorian on the client and submit them as ISO date-time strings, servers located in other timezones (e.g. UTC) may parse them incorrectly, shifting dates to the previous calendar day.
Solution: Send dates as pure calendar strings (YYYY-MM-DD) without time or timezone suffixes, or explicitly parse date strings in NST on the server.

6. Prioritize Keyboard Accessibility (a11y)

Ensure your date fields can be navigated by users relying solely on screen readers or keyboards.
Solution: Enable keyboardHelp: true and ensure target inputs are focusable. When the picker is open, users can navigate the monthly grid using the arrow keys, select a date by pressing Enter, and cancel/close the panel by pressing Escape.

7. Perform Server-Side Validation

Client-side validations, including input masking (autoMask: true) and boundaries (minDate, maxDate), are purely for user experience and can be bypassed.
Solution: Always parse, validate, and sanitize the submitted date value on your backend to ensure it falls within the absolute range bounds (1970–2100 BS) before saving.

8. Performance Tuning: Lazy Initialization

If your application renders hundreds of date-picker fields in a single scrollable grid or dashboard page, instantiating a class on every DOM node during initial page load can increase script execution time.
Solution: Dynamically initialize the datepicker instance only when the input field receives focus, and destroy it on blur, minimizing concurrent active DOM listeners.

futureOnly boolean false Restricts selections to future dates only (today and all past dates are disabled).

Instance Methods #

After initialization, you can save the returned instance variable and call API methods to programmatically manipulate the datepicker's state:

Method Name Arguments Returns Description
setDate(dateObj) { year, month, day } void Sets and highlights the currently selected date. Triggers onChange callbacks.
getDate() None Object Returns the current selected date details (returns single selected date, range boundaries, or multiple selections array).
clear() None void Resets the picker state, clears selected fields, and refreshes calendar grids.
setTheme(themeName) themeName (string) void Switches the visual picker layout theme dynamically (e.g. picker.setTheme('midnight')).
setLang(langCode) 'ne' | 'en' void Switches languages dynamically (e.g. picker.setLang('en')).
open() None void Opens the popup calendar overlay programmatically.
close() None void Closes the popup calendar overlay programmatically.
destroy() None void Removes calendar nodes, detaches form inputs event listeners, and purges references.
Method Invocation Example 
const picker = new NepaliDatePicker('#my-input');

// Set value to Baisakh 1, 2083 BS
picker.setDate({ year: 2083, month: 1, day: 1 });

// Change theme and language on the fly
picker.setTheme('neon-cyberpunk');
picker.setLang('en');

Event Callbacks #

The following event hooks can be configured to catch user selection events:

Callback Option Arguments Description
onChange date (Object | Array) Triggered when a date day is selected. In single mode, returns a formatted object. In multiple selection, returns an array of selected dates.
onRangeChange start (Obj), end (Obj) Only triggered in 'range' mode. Sends start boundary and end boundary date details respectively.
onMonthChange { year, month } Fired when navigating to a new month or year view.
onOpen None Fired immediately when the calendar popup opens.
onClose None Fired immediately when the calendar popup closes.
onToday date (Object) Fired when the shortcut "today" button is clicked in the calendar footer. Returns today's BS date object.
onClear None Fired when the "clear" button is clicked in the calendar footer.

Static Helpers #

The NepaliDatePicker class exposes static methods directly, allowing conversions or today checks without initializing DOM node instances:

Static Method Signature Returns Description
NepaliDatePicker.today() { year, month, day } Returns the current date in Bikram Sambat (BS).
NepaliDatePicker.bsToAd(y, m, d) { year, month, day, date } Converts a BS date numbers to AD. The date property is a standard JS Date object instance.
NepaliDatePicker.adToBs(jsDateObj) { year, month, day } Converts a Gregorian Date object to Bikram Sambat (BS).
Static Utility Examples 
// Get current BS date details
const today = NepaliDatePicker.today(); // e.g. { year: 2083, month: 3, day: 15 }

// Convert BS to AD
const adResult = NepaliDatePicker.bsToAd(2083, 3, 15);
console.log(adResult.date); // JS Date: Sun Jun 29 2026

// Convert AD to BS
const bsResult = NepaliDatePicker.adToBs(new Date());
console.log('BS Date details:', bsResult);

Calendar Limits #

Because Bikram Sambat is a hybrid solar-lunar calendar, the number of days in each month changes dynamically year-over-year. The library contains verified calendar metadata map configurations from the year १९७० वि.सं. to २१०० वि.सं. (1913 AD to 2043 AD).

Important Constraint
Any attempt to load or convert dates outside the 1970–2100 BS boundary will return conversion warnings and fallback to boundary values.

Frequently Asked Questions #

Does this library require jQuery or external dependencies?

No. Nepali DatePicker Studio is built using 100% pure vanilla JavaScript and CSS. It has zero external dependencies and runs completely independent of any framework.

Can I use this in modern frameworks like React, Vue, or Angular?

Yes. You can import the CSS/JS assets and initialize the picker inside your component lifecycle hooks (e.g., useEffect in React or onMounted in Vue) by passing the DOM input selector or target element reference.

How does the library handle Bikram Sambat year variations?

Since BS month lengths vary and are not mathematically constant like AD, the conversion relies on a built-in pre-mapped calendrical dataset. This ensures 100% conversion accuracy from 1970 BS up to 2100 BS.

How does the library handle timezone shifts during date conversion?

Calendar mapping and conversions are calculated using local integer representation fields (Year, Month, Day) rather than local browser time shifts, preventing common discrepancies introduced by JavaScript's native timezone parsing.

Can I validate input values on my server backend?

Yes. It is recommended to use the exportAdInput parameter. This automatically populates a hidden input with the corresponding Gregorian (AD) date format (YYYY-MM-DD) upon picker selections, which you can easily parse and validate on your backend.

How do I programmatically get or set a date?

You can call the instance methods. Call instance.setDate({ year: 2083, month: 2, day: 15 }) to update the value programmatically, or call instance.getSelectedDate() to extract the current selection object.

Why is my calendar tooltip or layout transparent or broken?

This occurs if the stylesheet file is not loaded. Ensure that dist/nepali-datepicker.css is loaded in the <head> tag of your document before initializing the date picker script.

Can I style my own theme?

Absolutely. All widget layouts are styled via standard CSS Custom Properties. You can easily override variables (e.g., --ndp-primary, --ndp-selected-bg) to match your application's design system.
